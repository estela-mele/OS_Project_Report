Game of Life Pattern File Analysis

File Format Specifications

The gen0 file format requirements:

  • Grid size: Exactly 10 lines × 10 characters
  • Live cells: Represented by period (.)
  • Dead cells: Represented by underscore (_)
  • Comments: Lines starting with # are ignored
  • Whitespace: Significant - maintains grid structure

Example Pattern File

# Sample gen0 file __.__..___ __.._.____ ____.___.. _._______. ____._____ ..__...___ ____._____ ___...____ __.._..___ _..___..__

Why Underscores Instead of Spaces?

The script uses underscores because:

  • Bash arrays split on whitespace by default
  • Underscores are preserved during array conversion
  • Spaces would collapse during processing
  • Alternative solution: Use different field separator (IFS)

Enhanced Format Suggestions

For better pattern files consider:

  • More visual characters (■ for live, space for dead)
  • Variable grid sizes
  • Pattern labeling
  • Multiple pattern support

Improved Example

# Glider pattern (period 4) ■ ■ ■ ■■

Creating Custom Patterns

Steps to create your own:

  1. Create new text file
  2. Make 10 lines of 10 characters each
  3. Use . for live, _ for dead cells
  4. Add comments with # prefix
  5. Save and use with life.sh

Interesting Pattern Ideas

  • Still lifes: Block, beehive, loaf
  • Oscillators: Blinker, toad, beacon
  • Spaceships: Glider, LWSS
  • Methuselahs: R-pentomino, diehard
